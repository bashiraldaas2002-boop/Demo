# استراتيجية توزيع الأحمال – Round Robin

## 1. المقدمة

تم تنفيذ آلية **توزيع الأحمال (Load Distribution)** ضمن مشروع نظام التجارة الإلكترونية عالي الأداء.
الهدف من هذه الآلية هو محاكاة توزيع الطلبات على أكثر من خادم بهدف تحسين الأداء وتقليل الضغط على خادم واحد.

---

## 2. الاستراتيجية المستخدمة

تم استخدام خوارزمية **Round Robin** لتوزيع الأحمال.

وهي طريقة بسيطة وعادلة تقوم على توزيع الطلبات بشكل دوري ومتتابع على مجموعة من الخوادم.

---

## 3. آلية العمل

بافتراض وجود ثلاثة خوادم وهمية:

* الخادم الأول (Server 1)
* الخادم الثاني (Server 2)
* الخادم الثالث (Server 3)

يتم توزيع الطلبات بالشكل التالي:

* الطلب 1 → الخادم 1
* الطلب 2 → الخادم 2
* الطلب 3 → الخادم 3
* الطلب 4 → الخادم 1
* الطلب 5 → الخادم 2
* الطلب 6 → الخادم 3

ويستمر هذا النمط بشكل دوري مع باقي الطلبات.

---

## 4. طريقة التنفيذ

تم تنفيذ خوارزمية Round Robin باستخدام عدّاد ذري (Atomic Counter) لضمان الأمان في بيئة متعددة الخيوط.

```java id="k2m9rr"
private AtomicInteger counter = new AtomicInteger(0);

public ExecutorService getNextServer() {
    int index = counter.getAndIncrement() % 3;

    return switch (index) {
        case 0 -> server1;
        case 1 -> server2;
        default -> server3;
    };
}
```

حيث يتم تمثيل كل خادم باستخدام `ExecutorService` لمحاكاة معالجة مستقلة لكل خادم.

---

## 5. مميزات Round Robin

* بسيط وسهل التنفيذ
* يوزع الطلبات بشكل عادل
* مناسب للخوادم ذات القدرة المتساوية
* مناسب لمحاكاة الأنظمة الموزعة

---

## 6. العيوب

* لا يأخذ بعين الاعتبار ضغط أو حمل الخادم
* قد يرسل طلبات ثقيلة إلى خادم غير مناسب
* غير مثالي للأنظمة الحقيقية الديناميكية

---

## 7. الخلاصة

تمكنت استراتيجية Round Robin من تحقيق توزيع فعال للأحمال بين عدة خوادم وهمية داخل المشروع، مما ساهم في تحسين الأداء ومحاكاة بيئة الأنظمة الموزعة بشكل واقعي.
